CAREER MODE IMPLEMENTATION PLAN

We are going to build Career Mode in small, safe increments. We will work feature-by-feature using the IDs below (F001, F002, ...).

For each feature:
- Make the smallest correct change that fully implements the feature.
- Reuse existing patterns and utilities from /engine and the current project structure (routing, contexts, hooks, UI patterns).
- Keep Career Mode free of debug UI.
- Keep state changes serializable and saveable via existing save/load systems.
- After each feature, ensure the app compiles and the main flow still works.
- Look inside the engine folder at the game engine component that already exist and use them when needed
- prime react and font awesome as well as react-router-dom are already installs and should be used if needed.

Do not implement future features early. Only do exactly the feature requested in the current prompt.



FEATURE F001: Home – Start Career entry

Requirements:
- Add a new button on the Home screen labeled "Start Career".
- Clicking it navigates into the Career Mode entry screen using the existing routing system.
- Career Mode should run in non-debug mode (set a context flag if gameContext already supports it, otherwise introduce a simple `mode: "career" | "sandbox"` field).
- Keep styling consistent with existing UI patterns/components.
- Do not remove the button to start a debug team selection, that is still needed and that functionality shouldn't be touched.

Deliverables:
- Home screen updated with the button.
- Navigation works to the Career entry route/screen.
- No debug UI should appear after entering Career mode.

Implementation notes:
- Reuse existing button component(s) if present.
- Do not implement StartCareer logic yet; if needed, create a minimal placeholder screen/component for the route.

FEATURE F005: Draft queue mechanic

  - Keep

- Candidate queue length is 4.
- Queue order must be random.
- If the player reaches candidate 4, Skip is disabled and they must Keep.
- UI should show queue position like "Candidate 2 of 4".

Deliverables:
- Queue system works across positions.
- Queue persists through step flow.



FEATURE F006: Draft candidate generation and rarity

Requirements:
- Replace placeholder generator with real generator.
- Typical candidate rating range: 68–74 overall.
- Rare candidate range: 75–80.
- Rare chance increases when the user skips players.

Example rule:
- Base rare chance: 5%
- Each skip adds +6% chance
- Max cap around 25%.

Players should have attributes:
- finishing
- passing
- control
- defending
- offBall
- workRate
- goalkeeping

Attribute distributions by role:
GK: high goalkeeping, low others
DEF: high defending/workRate
MID: high passing/control/offBall
FWD: high finishing/offBall

Deliverables:
- Draft candidates generated with correct rarity distribution.
- Skipping meaningfully changes rare appearance probability.
